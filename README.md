# Hospital Queue System

A full-stack hospital queue management platform built to improve patient appointment booking, doctor queue visibility, and hospital staff operations.

The system contains three connected applications:

- **Admin Dashboard** for hospital staff
- **Backend API** for business logic and data handling
- **Mobile App** for patients

This project is structured as a multi-application repository.

---

## 1. Project Purpose

Hospitals and clinics often face common queue management problems:

- Patients do not know when their turn will come.
- Reception staff manually manage appointments and waiting lists.
- Doctors may not have a clear view of upcoming patients.
- Patients spend unnecessary time waiting at the hospital.
- Appointment and queue information can become scattered across systems.

The goal of this project is to create a simple digital system where:

- Patients can book appointments from a mobile app.
- Patients can view live queue status.
- Admin staff can manage doctors and appointments.
- Hospital staff can monitor the current and upcoming patients.
- Backend APIs keep the system data consistent.
